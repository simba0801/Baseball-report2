## 1. 기존 UI 구조 분석 (Streamlit)

### 1.1 전체 레이아웃

**페이지 구성**
- **Sidebar**: 자동 축소 (`initial_sidebar_state="collapsed"`)
- **Header**: 제목 + 로그인 상태 표시
- **Main Area**: 화면별로 변경되는 콘텐츠 영역
- **Layout**: Wide (전체 폭 사용)

**시각적 특징**
- 배경색: 밝은 회색 (#f0f2f6)
- 주 색상: 파란색 (#1f77b4)
- 텍스트 크기: 크고 가독성 중심
- 이모지: 대량 사용 (네비게이션 및 상태 표시용)

### 1.2 화면별 UI 요소 분석

#### 화면 1: 홈 화면 (Home)

**구성**
```
┌─────────────────────────────────────────┐
│  ⚾ Baseball Report (헤더, 파란색)      │
├─────────────────────────────────────────┤
│ ✅ admin님이 로그인 중입니다 [로그아웃]  │ (우측 정렬)
├─────────────────────────────────────────┤
│ ### 메인 메뉴                            │
│                                         │
│ [🔐 로그인] [📝 선수등록] [📋 기록하기] [📊 결과조회]
│   (4개 버튼, 가로 배치, full-width)     │
├─────────────────────────────────────────┤
│ ### 📌 사용 가능한 기능                   │
│ - 로그인: 기존 사용자 로그인            │
│ - 선수등록: 이메일, ID, PW로 회원가입   │
│ - 기록하기: 로그인 후 경기 기록 입력    │
│ - 결과조회: 로그인 후 입력된 기록 조회  │
└─────────────────────────────────────────┘
```

**UI 요소**
- 헤더: st.markdown() 커스텀 HTML
- 로그인 표시: st.success() (초록색 배경)
- 버튼: st.button() (회색/흰색, 사각형 모서리)
- 설명 텍스트: st.markdown()

## 2. 화면 재설계 (Next.js)

### 2.1 페이지 구조 및 네비게이션

```typescript
// 라우팅 계층 구조
app/
  ├── (auth)/                    // 인증 필요 없음
  │   ├── login/
  │   ├── signup/
  │   └── logout/
  ├── (protected)/                // 인증 필요
  │   ├── home/
  │   ├── player/
  │   │   ├── register/
  │   │   └── edit/
  │   └── record/
  │       ├── input/
  │       └── results/
  └── middleware.ts              // 인증 검증
```

**네비게이션 흐름도**
```
로그인 전
  ├─→ /login              (로그인 페이지)
  ├─→ /signup             (회원가입 페이지)
  └─→ / (리다이렉트)

로그인 후
  ├─→ /home               (홈 대시보드)
  ├─→ /player/register    (선수 정보 등록)
  ├─→ /player/view        (선수 정보 보기)
  ├─→ /record/input       (경기 기록 입력) ⭐
  └─→ /record/results     (경기 결과 조회)
```

## 5. 인터랙션 설계

### 5.1 사용자 입력 흐름

**로그인 흐름**
```
1. 입력 (ID, 비밀번호)
   ↓
2. "로그인" 버튼 클릭 → 버튼 disabled, 로딩 spinner 표시
   ↓
3. API 호출 (POST /api/auth/login)
   ↓
4a. 성공 → 로컬스토리지에 토큰 저장
            → Context에 user_id 설정
            → /home으로 자동 리다이렉트
            → 토스트: "로그인 되었습니다"
   ↓
4b. 실패 → 폼 아래에 에러 메시지 표시 (빨간색)
            → 버튼 다시 활성화
            → 입력값 유지
```

**기록 입력 흐름**
````
1. 경기 정보 입력 (날짜, 상대팀, ...)
   ↓
2. 투구 위치 선택 (터치패드)
   ↓
3. "투구 기록하기" 클릭
   ↓
4. 게임 상태 업데이트
   - ball_count 또는 strike_count 증가
   - 현재 상황판 리렌더링
   ↓
5. 타자 액션 선택
   ↓
6. 결과 기록 클릭
   ↓
7. 게임 상태 재업데이트 (out_count, runs 등)
   ↓
8. 반복 (투구 추가) 또는 "경기 저장하기"
   ↓
9. API 호출 (POST /api/record/save)
   ↓
10. 성공 → 풍선 애니메이션 + 토스트
             → "새 게임 시작" 또는 "돌아가기" 버튼
```

### 5.2 버튼 클릭 동작

#### 경기 저장 버튼

```typescript
const handleSaveGame = async () => {
  // 1. 검증
  if (!opponent) {
    showToast({ type: "error", message: "상대팀을 입력해주세요" });
    return;
  }

    return;
  }

  // 2. 로딩 시작
  setLoading(true);

  try {
    // 3. API 호출
    const response = await api.post("/api/record/save", {
      date: gameDate,
      opponent,
      position,
      inning: currentInning,
      pitches,
      runs,
      earned_runs,
      errors
    });

    // 4. 성공 처리
    showToast({ type: "success", message: "경기 기록이 저장되었습니다!" });
    showConfetti();  // 풍선 효과
    
    // 5. 상태 초기화
    resetGameSession();
    
    // 6. 페이지 이동 (선택)
    setTimeout(() => router.push("/record/results"), 1000);

  } catch (error) {
    // 에러 처리

  } finally {
    setLoading(false);
  }
};
```

| # | 개선 사항 | Streamlit | Next.js | 효과 |
