## 💬 기능 제안 상세 설명

### 1. 투구 기록 수정 기능
**왜 필요한가?**
- 현장에서 빠르게 기록하다 보면 오입력이 발생할 가능성 높음
- 현재 삭제만 가능하여 전체 기록을 다시 입력해야 함

**구현 예시**
```python
# TAB 2에 "✏️ 투구 수정" 버튼 추가
# 선택한 투구의 데이터를 폼에 미리 채우고 수정 후 저장
```

---

### 2. 비교 분석 기능
**왜 필요한가?**
- 투수 간 성능 차이를 객관적으로 파악 필요
- 같은 투수의 경기별 성과 변화 추적 중요

**시각화 예시**
```
투수 A vs 투수 B
━━━━━━━━━━━━━━━
공의 종류 비율        스트라이크율       평균 속도
[막대 그래프]        [원형 그래프]      [꺾은선 그래프]
```

---

### 3. 히트맵 시각화 (Zone Map)
**왜 필요한가?**
- 투구 위치를 숫자로만 표현하면 이해하기 어려움
- 시각적 표현으로 컨트롤 패턴 직관적 파악 가능

**구현 예시**
```
Strike Zone Heatmap
┌───┬───┬───┐
│   │ ● │   │
├───┼───┼───┤
│   │ ● │ ● │
├───┼───┼───┤
│   │   │   │
└───┴───┴───┘

색상: 점점 진해질수록 투구 빈도 높음
```

---

### 4. 대량 데이터 내보내기
**왜 필요한가?**
- Excel에서 추가 분석, 보고서 작성 필요성
- 외부 통계 프로그램과 연동 필요

**지원 형식**
- CSV (모든 경기 투구 데이터)
- Excel (탭별 정리된 데이터)
- PDF (보고서 형식)

---

### 5. 자동 보고서 생성
**왜 필요한가?**
- 코치·감독·부모에게 성과 전달 필요
- 정기적인 평가 자료로 활용

**보고서 포함 내용**
- 경기 요약 (날짜, 상대팀, 이닝 수)
- 투구 통계 (총 투구, 스트라이크, 볼, 율)
- 공의 종류 분석 (선호 공, 효과적인 공)
- 개선점 (컨트롤, 속도, 다양성)

---

### 6. 사용자 계정 및 팀 관리
**왜 필요한가?**
- 여러 투수 데이터를 조직적으로 관리 필요
- 팀원 간 데이터 공유로 협업 강화

**주요 기능**
- 회원가입 / 로그인
- 팀 생성 / 관리
- 투수 추가 / 관리
- 팀원 초대
- 데이터 공유 권한 설정
