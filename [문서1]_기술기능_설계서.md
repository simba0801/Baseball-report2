## 1. AS-IS 분석

### 1.1 핵심 기능 (5개)

1. **사용자 인증 시스템**
   - 회원가입 (이메일, ID, 비밀번호)
   - 로그인 (ID/비밀번호 검증)
   - 로그아웃 (admin 제외)
   - Admin 자동 로그인

2. **선수 정보 관리**
   - 선수 신체정보 (키, 체중)
   - 학력 정보 (초등/중등/고등학교 입학/졸업년도, 팀명)
   - 사용자별 단일 선수 정보 저장

3. **경기 기록 입력**
   - 투수 기본정보 (경기 날짜, 상대팀, 등판 구분)
   - 게임 진행 상황 (이닝, 타순, 아웃카운트, 잔루, 스트라이크/볼 카운트)
   - 투구 기록 (구종, 위치/Zone, 특수상황)
   - 타자 결과 (스윙, 파울, 안타, 에러 등)

4. **경기 기록 조회 및 통계**
   - 선수별 총 경기수, 총 투구수, 실점/자책점
   - 경기별 상세 투구 기록
   - 투구 위치별 분포 (Zone 1-25)
   - 구종별 투구 횟수

5. **데이터 저장 및 관리**
   - JSON 기반 파일 저장 (users.json, players.json, records.json)
   - 사용자별 데이터 격리
   - 경기 기록의 순차적 저장

### 1.2 사용자 흐름

```
홈화면 (game_state: "home")
    ├─→ 로그인 화면 → 로그인 성공 → 홈화면 (logged_in_user 설정)
    ├─→ 회원가입 화면 → 가입 완료 → 선수등록 화면 → 홈화면
    ├─→ 선수등록 화면 (로그인 필수)
    ├─→ 기록하기 화면 (로그인 필수)
    │    └─→ 경기정보 입력 → 투구 기록 입력 (반복) → 타자 액션 → 경기 저장
    └─→ 결과조회 화면 (로그인 필수)
         └─→ 경기별 상세 통계 조회
```

### 1.3 상태 관리 (st.session_state 구조)

```python
st.session_state = {
    # 화면 네비게이션
    "current_screen": str,  # 현재 화면 식별자
                            # "home", "login", "signup", "player_register", 
                            # "record_input", "results"
    
    # 사용자 인증
    "logged_in_user": str | None,  # 로그인한 사용자 ID
    "user_id": str | None,          # 현재 사용자 ID (logged_in_user와 동일)
    
    # 경기 진행 상태 (game_session)
    "game_session": {
        "date": datetime,                    # 경기 날짜
        "opponent": str,                     # 상대팀명
        "position": str,                     # "선발", "중간", "마무리"
        "current_inning": int,               # 1-9 이닝
        "current_batter_order": int,         # 1-9 타순
        "batter_hand": str,                  # "좌" 또는 "우"
        "out_count": int,                    # 0-2 아웃
        "ball_count": int,                   # 0-3 볼
        "strike_count": int,                 # 0-2 스트라이크
        "bases": {
            "1루": bool,
            "2루": bool,
            "3루": bool
        },
        "runners_home": int,                 # 홈에 들어온 주자 수
        "pitches": [                         # 투구 기록 배열
            {
                "pitch_num": int,
                "pitch_type": str,           # "직구", "커브" 등
                "result": str,               # "Strike" 또는 "Ball"
                "location": int,             # Zone 1-25
                "special_situation": str,    # "폭투", "패스드볼" 등
                "ball_count": int,
                "strike_count": int,
                "out_count": int
            }
        ],
        "runs": int,                         # 총 실점
        "earned_runs": int,                  # 자책점
        "errors": int                        # 에러 수
    },
    
    # 선택 상태
    "selected_zone": int | None  # 선택된 터치패드 존 (1-25)
}
```

### 1.4 데이터 흐름

```
입력 → Streamlit 위젯 
    → st.session_state 업데이트
    → 함수 호출 (save_users, save_players, save_records)
    → JSON 파일 저장
    ↓
JSON 파일 (로컬 디스크)
    ↓
조회 → load_* 함수 호출
    → JSON 파일 읽기
    → st.session_state에 로드
    → UI에 렌더링
```

**데이터 구조 (JSON 포맷)**

| 파일명 | 구조 | 용도 |
|--------|------|------|
| `users.json` | `{ [user_id]: { email, password, created_date } }` | 사용자 계정 관리 |
| `players.json` | `{ [user_id]: { name, height, weight, 학력정보, created_date } }` | 선수 신체/학력 정보 |
| `records.json` | `{ [user_id]: [ { date, opponent, pitches[], runs, errors, ... } ] }` | 경기 기록 (배열) |

### 2.3 역할 분리 (FE vs BE)

| 작업 | Streamlit (현재) | Next.js FE | Backend API |
|------|-----------------|-----------|-------------|
| UI 렌더링 | ✓ | ✓ | - |
| 폼 입력 검증 | ✓ (간단함) | ✓ (상세) | ✓ (최종 검증) |

| 비즈니스 로직 | 일부 | 최소화 | ✓ (대부분) |
| 데이터 저장 | JSON 파일 | - | ✓ (DB) |
| 세션 관리 | Streamlit 자체 | localStorage/sessionStorage | ✓ (쿠키/JWT) |

### 2.4 통신 방식

```

  headers: { Authorization: "Bearer {token}" }
  response: PlayerSchema
```

```
POST /api/record/save
  headers: { Authorization: "Bearer {token}" }
  request: GameRecordSchema (date, opponent, pitches[], ...)

GET /api/record?date_from=...&date_to=...
  headers: { Authorization: "Bearer {token}" }
  response: [ GameRecordSchema ]

DELETE /api/record/{record_id}
  headers: { Authorization: "Bearer {token}" }
  response: { message }

GET /api/record/stats
  headers: { Authorization: "Bearer {token}" }
  response: { total_games, total_pitches, avg_speed, ... }
```

## 3. API 명세

### 3.1 인증 API

#### 3.1.1 회원가입

**Endpoint**: `POST /api/auth/signup`

**Request**
```typescript
{
  email: string;           // 이메일 (유니크 검증)
  user_id: string;         // 사용자 ID (영문, 숫자, 언더스코어만, 3-20자)
  password: string;        // 비밀번호 (최소 6자)
}
```

**Response (Success - 201)**
```typescript
{
  success: true;
  token: string;           // JWT 토큰 (유효기간: 7일)
  user_id: string;
  message: string;         // "회원가입이 완료되었습니다"
}
```

**Error Cases**
| HTTP 상태 | 에러 코드 | 메시지 |
|----------|---------|--------|
| 400 | INVALID_INPUT | "필드를 모두 입력해주세요" |
| 400 | PASSWORD_MISMATCH | "비밀번호가 일치하지 않습니다" |
| 409 | DUPLICATE_USER_ID | "이미 존재하는 ID입니다" |
| 409 | DUPLICATE_EMAIL | "이미 등록된 이메일입니다" |
| 500 | SERVER_ERROR | "서버 오류 발생" |

---

#### 3.1.2 로그인

**Endpoint**: `POST /api/auth/login`

**Request**
```typescript
{
  user_id: string;
  password: string;
}
```

**Response (Success - 200)**
```typescript
{
  success: true;

  message: string;         // "{user_id}님이 로그인했습니다"
```

**Error Cases**

| 401 | INVALID_PASSWORD | "비밀번호가 일치하지 않습니다" |

### 3.2 선수 정보 API

#### 3.2.1 선수 정보 등록/수정

**Endpoint**: `POST /api/player/register`

**Headers**
```
Authorization: Bearer {token}
Content-Type: application/json
```

**Request**
```typescript
{
  name: string;                    // 선수명 (필수)
  height: number;                  // 키 (cm, 150-220)
  weight: number;                  // 몸무게 (kg, 50-150)
  elementary_graduation: number;   // 초등 졸업연도
  elementary_team: string;         // 초등 팀명
  middle_graduation: number;       // 중등 졸업연도
  middle_team: string;             // 중등 팀명
  high_school: string;             // 고등학교명 (필수)
  high_grade: number;              // 학년 (1-3)
  high_graduation: number;         // 졸업연도
}
```

**Response (Success - 201)**
```typescript
{
  success: true;
  player_id: string;
  message: string;                 // "선수 정보가 저장되었습니다"
}
```

**Error Cases**
| HTTP 상태 | 에러 코드 | 메시지 |
|----------|---------|--------|
| 400 | MISSING_REQUIRED | "필수 필드: name, high_school" |
| 400 | INVALID_HEIGHT | "키는 150-220cm 범위입니다" |
| 400 | INVALID_WEIGHT | "몸무게는 50-150kg 범위입니다" |
| 401 | UNAUTHORIZED | "인증이 필요합니다" |

---

#### 3.2.2 선수 정보 조회

**Endpoint**: `GET /api/player`

## 주요 주의사항

### 보안
1. **비밀번호**: bcrypt hashing (최소 10 rounds)
2. **JWT**: HttpOnly 쿠키에 저장, CSRF 토큰 사용
3. **SQL Injection**: ORM(SQLAlchemy) 사용 필수
4. **CORS**: 명시적으로 설정 (와일드카드 금지)
5. **Rate Limiting**: API에 적용

### 성능
1. **DB 인덱싱**: user_id, created_date에 인덱스 설정
2. **API 응답 캐싱**: 5분 정도의 staleTime 설정
3. **무한 스크롤**: 경기 기록 조회 시 pagination 필수
4. **이미지 최적화**: next/image 사용

### 사용자 경험
1. **로딩 상태**: 모든 비동기 작업에 spinner 표시
2. **에러 메시지**: 구체적이고 actionable한 메시지
3. **Optimistic UI**: 폼 제출 시 즉시 반영
4. **Undo 기능**: 경기 기록 삭제 시 5초 대기 (선택사항)

### 데이터 이관
- 기존 JSON 파일 → PostgreSQL로 한 번에 마이그레이션
- 마이그레이션 스크립트 작성 필수
- 데이터 검증 로직 포함
